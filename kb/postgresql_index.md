# PostgreSQL 인덱스

인덱스는 테이블의 특정 컬럼에 대한 조회 속도를 높이기 위한 자료구조로, 기본적으로 B-Tree 구조를 사용한다. 인덱스가 없으면 조건에 맞는 행을 찾기 위해 테이블 전체를 순차 스캔해야 하지만, 인덱스가 있으면 정렬된 구조를 통해 빠르게 원하는 행에 접근할 수 있다.

다만 인덱스는 조회 성능을 높이는 대신 쓰기(INSERT, UPDATE, DELETE) 성능에는 부담을 준다. 데이터가 변경될 때마다 인덱스도 함께 갱신해야 하기 때문이다. 그래서 자주 조회되지만 쓰기가 드문 컬럼에 인덱스를 거는 것이 일반적인 원칙이다.
