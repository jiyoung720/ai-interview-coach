# 세션 기반 인증 vs 토큰 기반 인증

세션 기반 인증은 사용자가 로그인하면 서버가 세션 정보를 서버 메모리나 별도 저장소(Redis 등)에 저장하고, 클라이언트에는 세션 ID만 담긴 쿠키를 전달하는 방식이다. 이후 요청마다 그 세션 ID로 서버에 저장된 정보를 조회해 사용자를 식별한다.
이 방식의 특징은 서버가 상태를 직접 들고 있다는 점이다. 그래서 서버 쪽에서 특정 세션을 즉시 무효화하기가 쉽다. 다만 서버가 여러 대로 확장될 경우, 모든 서버가 세션 저장소를 공유해야 하는 부담이 생긴다.
토큰 기반 인증(JWT 등)은 반대로 서버가 상태를 저장하지 않는(stateless) 방식이다. 사용자 정보가 토큰 자체에 서명된 채로 담겨 있어, 서버는 매 요청마다 서명만 검증하면 되고 별도의 저장소 조회가 필요 없다.
이 차이 때문에 세션 방식은 수평 확장 시 별도의 공유 저장소가 필요한 반면, 토큰 방식은 서버가 늘어나도 별도 동기화 없이 확장이 쉽다. 다만 토큰 방식은 발급된 토큰을 서버가 강제로 무효화하기 어렵다는 단점이 있어, 만료 시간을 짧게 잡고 재발급 절차를 두는 방식으로 보완한다.
