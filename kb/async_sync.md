# 동기 처리와 비동기 처리

동기 처리는 하나의 작업이 끝나야 다음 작업이 시작되는 방식이다. 예를 들어 DB 조회 결과를 기다리는 동안 그 스레드는 아무 일도 못 하고 대기하게 된다. 코드 흐름이 직관적이라는 장점이 있지만, I/O 대기 시간이 길어질수록 자원 낭비가 커진다.
비동기 처리는 작업을 시작해두고 그 결과를 기다리지 않은 채 다른 작업을 계속 처리하다가, 결과가 준비되면 그때 처리하는 방식이다. Python에서는 async와 await 키워드와 이벤트 루프를 통해 구현된다.
이벤트 루프는 단일 스레드에서 여러 작업을 번갈아 처리하는 구조다. I/O 대기가 발생하면 그 작업을 잠시 미뤄두고 다른 대기 중인 작업을 실행한 뒤, I/O가 끝나면 다시 돌아와 이어서 처리한다. 여러 스레드를 만드는 것이 아니라, 하나의 스레드가 대기 시간을 효율적으로 활용하는 방식이다.
비동기 처리는 I/O 바운드 작업에서 큰 이점을 준다. 반면 CPU 바운드 작업에는 비동기가 자동으로 이점을 주지 않는데, 이벤트 루프가 하나의 스레드로 동작하기 때문에 무거운 연산이 그 스레드를 계속 점유하면 다른 작업이 전혀 처리되지 못하기 때문이다.
