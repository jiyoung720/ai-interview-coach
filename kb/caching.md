# 캐싱 전략

캐싱은 자주 조회되지만 자주 바뀌지 않는 데이터를 빠른 저장소(메모리 등)에 미리 저장해두어, 매번 원본 데이터 소스(DB 등)에 접근하지 않고도 빠르게 응답하기 위한 기법이다. Redis나 Memcached 같은 인메모리 저장소가 대표적으로 사용되며, JWT 블랙리스트나 세션 저장소로도 Redis가 흔히 쓰인다.

캐시 무효화(Cache Invalidation)는 원본 데이터가 변경됐을 때 캐시된 값도 함께 갱신하거나 삭제하는 것을 말한다. 캐시 무효화를 제대로 처리하지 않으면 사용자가 오래된(stale) 데이터를 보게 되는 문제가 생긴다. "캐시 무효화는 컴퓨터 과학에서 가장 어려운 문제 중 하나"라는 말이 있을 정도로 신중한 설계가 필요하다.

TTL(Time To Live)은 캐시된 데이터가 유효한 시간을 정하는 방식으로, 일정 시간이 지나면 자동으로 캐시가 만료되어 다시 원본에서 데이터를 가져오게 한다. 무효화 로직을 직접 구현하지 않아도 되는 간단한 방법이지만, TTL이 너무 길면 오래된 데이터를 보여줄 위험이, 너무 짧으면 캐시 효과가 줄어드는 트레이드오프가 있다.

캐시 적중률(Cache Hit Ratio)은 전체 요청 중 캐시에서 바로 응답한 비율을 말하며, 이 값이 높을수록 캐싱 전략이 효과적이라는 뜻이다. 반대로 캐시에 없어서 원본을 조회해야 하는 경우를 Cache Miss라고 한다.
