# Docker

Docker는 애플리케이션과 그 실행 환경(라이브러리, 설정 등)을 하나의 컨테이너로 패키징하는 기술이다. 가상머신(VM)이 하드웨어 자체를 가상화해 각각 별도의 OS를 띄우는 것과 달리, 컨테이너는 호스트 OS의 커널을 공유하면서 프로세스 수준에서 격리되기 때문에 훨씬 가볍고 빠르게 실행된다.
이미지는 컨테이너 실행에 필요한 파일 시스템과 설정을 담은 템플릿이며, 컨테이너는 그 이미지를 기반으로 실행되는 실제 인스턴스다. 이미지는 여러 개의 레이어로 구성되는데, 각 레이어는 Dockerfile의 한 명령어에 대응하며 캐싱되어 재빌드 시 변경되지 않은 레이어는 재사용된다.
Dockerfile은 이미지를 빌드하기 위한 명령어 집합을 담은 텍스트 파일이다. FROM으로 베이스 이미지를 지정하고, COPY나 RUN으로 파일 복사와 명령 실행을 정의하며, CMD나 ENTRYPOINT로 컨테이너 실행 시 동작할 명령을 지정한다.
여러 컨테이너를 함께 운영해야 할 때는 Docker Compose를 사용해 docker-compose.yml 파일 하나로 여러 서비스의 관계와 설정을 한 번에 정의하고 실행할 수 있다.
