# PostgreSQL — 인덱스와 트랜잭션

인덱스는 테이블의 특정 컬럼에 대한 조회 속도를 높이기 위한 자료구조로, 기본적으로 B-Tree 구조를 사용한다. 인덱스가 없으면 조건에 맞는 행을 찾기 위해 테이블 전체를 순차 스캔해야 하지만, 인덱스가 있으면 정렬된 구조를 통해 빠르게 원하는 행에 접근할 수 있다.

다만 인덱스는 조회 성능을 높이는 대신 쓰기(INSERT, UPDATE, DELETE) 성능에는 부담을 준다. 데이터가 변경될 때마다 인덱스도 함께 갱신해야 하기 때문이다. 그래서 자주 조회되지만 쓰기가 드문 컬럼에 인덱스를 거는 것이 일반적인 원칙이다.

트랜잭션은 여러 개의 SQL 작업을 하나의 논리적 단위로 묶는 것으로, ACID(원자성, 일관성, 고립성, 지속성) 속성을 보장한다. 트랜잭션 내의 모든 작업이 성공하면 COMMIT으로 반영되고, 하나라도 실패하면 ROLLBACK으로 전체가 취소되어 데이터 일관성이 깨지지 않는다.

트랜잭션 고립 수준(Isolation Level)은 동시에 실행되는 트랜잭션들이 서로에게 미치는 영향의 정도를 정의한다. PostgreSQL의 기본 고립 수준은 Read Committed이며, 더 엄격한 Serializable로 갈수록 동시성은 떨어지지만 데이터 정합성은 높아진다.
